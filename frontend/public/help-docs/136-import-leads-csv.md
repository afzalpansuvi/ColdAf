# How to Import Leads from a CSV File in ColdAF

**Last Update:** July 2026

**Post ID:** 136

Getting your contact list into ColdAF is the first step toward scaling your cold outreach. Whether you are migrating from a spreadsheet, exporting from a CRM, or working with a purchased list, learning how to **import leads csv coldaf** correctly will save you hours of manual data entry. This guide walks through the **csv upload cold email leads** process step by step, so you can start **bulk import cold outreach contacts** quickly and without errors. A clean import means better deliverability, cleaner targeting, and more accurate analytics down the road.

## You can import leads from a CSV file in just a few steps. Here's how to do it:

1. Log in to ColdAF and go to the **Leads** page from the main navigation.
2. Click the **Import Leads** button near the top of the page.
3. Select **CSV Upload** from the import options presented.
4. Click **Choose File** and select the CSV file from your computer.
5. Review the data preview that appears. ColdAF will show the first 10 rows so you can verify the file looks correct.
6. Map your CSV columns to ColdAF fields. For example, map **First Name** to **firstName**, **Last Name** to **lastName**, **Email** to **email**, **Company** to **company**, and **Title** to **title**.
7. Check the data preview again after mapping to confirm everything lines up properly.
8. Confirm the column mapping and click **Import** to start the upload.
9. Wait for the import to complete. The import status will show progress and any errors.
10. Review the import results page to see how many leads were added, skipped, or failed.

That's it! Now you can **import leads csv coldaf** in bulk and start building campaigns immediately.

## Notes

- Your CSV must include a header row. ColdAF uses the first row to identify columns automatically.
- The **Email** column is required. Any row without a valid email address will be rejected.
- Duplicate emails are flagged during import. You can choose to skip duplicates or update existing records.
- The maximum limit is 10,000 rows per import. If your list is larger, split it into multiple files.
- UTF-8 encoding is recommended to avoid character issues with international names or special symbols.
- The preview shows the first 10 rows for verification before you commit.
- Failed rows display specific error reasons so you can fix and re-import them.
